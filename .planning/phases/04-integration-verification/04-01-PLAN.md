  <action>
Run the following four grep scans in order from the repo root. Each scan must return **empty output** (non-zero exit = no matches = PASS). If any scan returns matches, report them verbatim and mark Task 1 as FAILED.

**Scan A — No forbidden tag EMISSION via println!/eprintln!:**
```
grep -rn 'println!\|eprintln!' vmproxy/src/ anylinuxfs/src/ \
  | grep '<anylinuxfs-' \
  | grep -v 'vmproxy-ready'
```
Expected: empty. Explanation: vmproxy/src/main.rs:1123 has `println!("<anylinuxfs-vmproxy-ready>")` which is filtered out by the `-v vmproxy-ready` exclusion. Any other match is a regression.

**Scan B — No forbidden tag STRING LITERALS in parsing/matching code:**
```
grep -rn \
  '"<anylinuxfs-type:\|"<anylinuxfs-label:\|"<anylinuxfs-nfs-export:\|"<anylinuxfs-mount:\|"<anylinuxfs-passphrase-prompt:\|"<anylinuxfs-force-output:\|"<anylinuxfs-exit-code:' \
  anylinuxfs/src/ vmproxy/src/
```
Expected: empty. These seven prefixes correspond to the nine VmEvent variants that replaced them; none should appear as string literals in production code.

**Scan C — parse_vm_tag_value fully deleted:**
```
grep -rn 'parse_vm_tag_value' anylinuxfs/src/ vmproxy/src/ common-utils/src/
```
Expected: empty. Phase 3 deleted this function from anylinuxfs/src/main.rs and removed its import from cmd_mount.rs.

**Scan D — Only vmproxy-ready on stdout (vmproxy):**
```
grep -n 'println!\|eprintln!' vmproxy/src/main.rs | grep -v 'vmproxy-ready'
```
Expected: empty or lines that do NOT contain `<anylinuxfs-`. If any `println!` line contains `<anylinuxfs-` without `vmproxy-ready`, that is a regression.

**If all four scans return empty**: Task 1 PASSES — record exit codes (all should be 1 from grep finding no matches, or 0 from the pipe returning nothing).
**If any scan returns output**: Task 1 FAILS — paste the offending lines into the summary and halt.
